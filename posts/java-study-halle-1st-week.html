JVM이란 무엇인가?

자바를 실행하기 위한 프로그램. JVM은 자바 프로그램을 실행할 때, Java를 Java bytecode(.class 파일)로 컴파일(명령어는 javac) 한 후 실행하며, 컴파일된 바이트코드는 java <arguments> <program name> 명령어로 실행한다. JVM은 컴파일된 바이트 코드를 인터프리터를 이용해 런타임에 instruction(= machine code)으로 해석하면서 프로그램을 실행하며 JIT 컴파일러를 이용해서 인터프리터 대신에 바이트 코드를 한번에 machine code로 변환해서 최적화하기도 한다.

JVM의 설계 목표이자 특징

  • 어플리케이션 코드가 돌아갈 수 있는 컨테이너 역할
    • jvm + bytecode interpreter
  • C/C++과는 다르게 안전한(메모리 관리 등) 실행환경 제공
    • 직접적인 메모리 참조가 불가능하며, bytecode verification을 제공한다.
  • 프로그래머가 신경쓰지 않아도 되는 메모리 관리.
  • 크로스 플랫폼 실행 환경 제공 (Write Once Run Anywhere)
    • class 파일로 변환된 파일은 어떤 OS(macOS, windows, linux …)이든 실행가능

JIT Compiler

런타임에 더 자주 실행되는 부분(hot spot)을 찾아서 최적화하기 위한 컴파일러. 매번 bytecode를 machine code로 해석하지 않고, 해석해둔 코드를 계속 사용하거나 코드 자체를 최적화한다.

바이트코드

javac 컴파일러가 java를 컴파일해서 내놓는 중간 결과물. .class파일이며 JVM이 이 파일을 런타임에 해석하면서 실행한다. 명령어 종류를 나타내는 opcode가 1바이트라 바이트코드라 부른다. 대략 200개 정도 사용 중. Bytecode는 플랫폼(or 머신)에 종속적이지 않으며 항상 동일한 포맷이다. Endian의 경우도 마찬가지인데, 어떤 플랫폼에서 생성되었던 간에, Big Endian(0x1234의 경우 12 34 로 저장하는 식. Little endian은 34 12 이다.)을 따른다.

컴파일/실행 하는 방법

컴파일

java 컴파일러는 JDK(Java Development Kit)에 들어있다.

javac 명령어로 컴파일할 수 있고, 컴파일 결과는 .class 파일로 구성된 자바 바이트코드 파일이다. 컴파일할 자바 버전을 지정할 수도 있는 등 여러 옵션을 제공한다.

$ javac [-source <컴파일할 소스 코드의 자바 버전>] [-target <컴파일될 코드의 자바 지원 버전>] ...

컴파일된 바이트코드는 javap -c <바이트코드 파일경로>명령어로 확인해볼 수 있다.

실행

java 명령어로 컴파일된 바이트코드를 실행할 수 있다.

JDK(Java Development Kit) vs JRE(Java Runtime Environment)

JRE는 말그대로 실행하기 위해 필요한 것들만 분리해둔 것이라 JRE가 JDK안에 속해있다. JDK에는 JRE뿐만 아니라 컴파일러, 디버거 등 개발에 필요한 도구들이 포함되어있다. Java 9 버전 부터는 JRE를 따로 제공하지는 않는다고 한다
